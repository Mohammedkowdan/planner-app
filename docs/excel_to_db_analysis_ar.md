# تحليل ملف الإكسل وتحويله إلى تصميم قاعدة بيانات

## 1) ملاحظات مهمة بعد فحص الملف
- الملف يحتوي على **15 ورقة**:
  - **14 ورقة متشابهة** بأسماء رقمية من `1` إلى `14`
  - ورقة واحدة باسم **`الإجماليات`**
- الأوراق `1` إلى `14` ليست جداول بيانات خام، بل **نماذج إدخال/تقرير** متشابهة.
- كل ورقة تمثل غالبًا **مشروعًا واحدًا** أو **نموذج مشروع**، لأن فيها حقل:
  - `اسم المشروع`
  - `المانح`
  - `نبذة عن المشروع`
  - `مخرجات المشروع`
  - `تاريخ البدء`
  - `تاريخ الانتهاء`
  - `الحالة`
  - `موازنة المشروع بالدولار`
- الصفوف من `5` إلى `26` تمثل **مواقع التنفيذ** حسب المحافظات، مع مؤشرات للمستفيدين.
- ورقة **الإجماليات** تجمع أرقام المحافظات من الأوراق `1` إلى `14` عن طريق المعادلات.

## 2) أهم نتيجة
الملف الحالي لا ينبغي تحويله مباشرةً كما هو إلى قاعدة بيانات على شكل:
- ورقة 1 = جدول
- ورقة 2 = جدول
- ...

هذا سيكون تصميمًا ضعيفًا وغير مرن.

**التصميم الصحيح** هو استخراج الكيانات الحقيقية التالية:
1. المشاريع
2. القطاعات
3. المانحون
4. المحافظات
5. المديريات
6. توزيع المشروع جغرافيًا
7. إحصاءات المستفيدين لكل مشروع/موقع
8. التقارير السنوية أو دورات التقرير

---

## 3) الحقول الظاهرة في نموذج المشروع
من خلال الأوراق المتكررة، الحقول الأساسية هي:

### بيانات المشروع
- اسم المشروع
- القطاع
- المانح
- نبذة عن المشروع
- مخرجات المشروع
- تاريخ البدء
- تاريخ الانتهاء
- الحالة:
  - منتهي
  - مستمر
- موازنة المشروع بالدولار

### بيانات التوزيع الجغرافي والتنفيذي
- المحافظة
- المديرية

### بيانات المستفيدين
- عدد الأسر
- طفل
- طفلة
- ذكور
- إناث
- الإجمالي
- إجمالي الذكور والأطفال
- إجمالي الإناث والطفلات

> ملاحظة: الحقول التالية **مشتقة** ويجب عدم تخزينها غالبًا كبيانات أصلية:
- الإجمالي = طفل + طفلة + ذكور + إناث
- إجمالي الذكور والأطفال = طفل + ذكور
- إجمالي الإناث والطفلات = طفلة + إناث

---

## 4) التصميم المقترح لقاعدة البيانات

### A. جدول القطاعات
يحفظ اسم القطاع الذي ينتمي إليه المشروع.

**sectors**
- id
- name
- code
- notes

### B. جدول المانحين
**donors**
- id
- name
- contact_info (اختياري)
- notes

### C. جدول المحافظات
**governorates**
- id
- name

### D. جدول المديريات
**districts**
- id
- governorate_id
- name

### E. جدول التقارير السنوية
يفيد إذا أردت إدارة التقارير حسب السنة أو الجهة.

**annual_reports**
- id
- title
- report_year
- organization_name
- notes

### F. جدول المشاريع
هذا أهم جدول.

**projects**
- id
- annual_report_id
- sector_id
- donor_id
- sheet_no  ← رقم الورقة الأصلية (1 إلى 14) للاحتفاظ بأثر المصدر
- project_name
- project_brief
- project_outputs
- start_date
- end_date
- status
- budget_usd
- created_at
- updated_at

### G. جدول مواقع تنفيذ المشروع
لأن المشروع قد ينفذ في أكثر من محافظة/مديرية.

**project_locations**
- id
- project_id
- governorate_id
- district_id (اختياري إذا لم توجد مديرية)
- families_count
- boys_count
- girls_count
- men_count
- women_count

> ويمكن إنشاء القيم المشتقة في View أو أثناء الاستعلام:
- total_beneficiaries
- total_males_with_boys
- total_females_with_girls

---

## 5) لماذا هذا التصميم هو الأفضل؟
لأن نفس المشروع قد:
- ينتمي إلى **قطاع واحد**
- يموله **مانح واحد** أو أكثر مستقبلًا
- يُنفذ في **عدة محافظات ومديريات**
- له **إحصاءات مختلفة لكل موقع**

وبالتالي فإن فصل المشروع عن الموقع الجغرافي وعن الإحصاءات يجعل قاعدة البيانات:
- أنظف
- أسهل للاستعلام
- قابلة للتوسع
- مناسبة لبناء لوحة تحكم وتقارير لاحقًا

---

## 6) العلاقات بين الجداول
- `projects.sector_id -> sectors.id`
- `projects.donor_id -> donors.id`
- `projects.annual_report_id -> annual_reports.id`
- `districts.governorate_id -> governorates.id`
- `project_locations.project_id -> projects.id`
- `project_locations.governorate_id -> governorates.id`
- `project_locations.district_id -> districts.id`

---

## 7) كيف نقرأ الإكسل ونُدخله في القاعدة؟
### على مستوى كل ورقة من 1 إلى 14:
1. نقرأ بيانات رأس الورقة:
   - اسم المشروع
   - القطاع
   - المانح
   - النبذة
   - المخرجات
   - التواريخ
   - الحالة
   - الموازنة
2. ننشئ سجلًا في جدول `projects`
3. نقرأ الصفوف من 5 إلى 26
4. لكل صف فيه محافظة أو بيانات فعلية:
   - نحدد المحافظة
   - نحدد المديرية إن وجدت
   - نحفظ أعداد الأسر والمستفيدين في `project_locations`

### ورقة الإجماليات
لا تحتاج إلى تخزين مباشر غالبًا، لأننا نستطيع **إنتاجها بالاستعلامات** من قاعدة البيانات.

---

## 8) ملاحظة حرجة جدًا
الملف الذي تم رفعه يبدو أنه يحتوي على:
- بنية التقرير
- أسماء المحافظات
- معادلات التجميع

لكن **معظم الحقول النصية والرقمية الفعلية فارغة** في الأوراق المتكررة.

هذا يعني:
- يمكنني بناء **تصميم قاعدة بيانات ممتاز**
- لكن لا يمكنني استخراج **بيانات مشاريع حقيقية مكتملة** من هذا الملف وحده إلا إذا كانت هناك نسخة مملوءة فعلاً

---

## 9) قرار التخزين: ماذا نخزن وماذا نحسب؟
### نخزن
- عدد الأسر
- طفل
- طفلة
- ذكور
- إناث
- المحافظة
- المديرية
- المشروع
- التواريخ
- الحالة
- الموازنة

### لا نخزن مباشرةً إلا عند الحاجة
- الإجمالي
- إجمالي الذكور والأطفال
- إجمالي الإناث والطفلات
- إجماليات التقرير السنوي

لأن هذه يمكن حسابها بسهولة من البيانات الأصلية.

---

## 10) مثال استعلامات مستقبلية ممكنة
- إجمالي المستفيدين لكل مشروع
- إجمالي المستفيدين لكل محافظة
- إجمالي المشاريع حسب القطاع
- إجمالي المشاريع المستمرة مقابل المنتهية
- مجموع الموازنة حسب المانح
- تقرير سنوي مطابق تمامًا لورقة `الإجماليات`

---

## 11) الخلاصة التنفيذية
أفضل تحويل لهذا الملف إلى قاعدة بيانات هو:

- **جدول مشاريع**
- **جدول قطاعات**
- **جدول مانحين**
- **جدول محافظات**
- **جدول مديريات**
- **جدول مواقع تنفيذ المشروع وإحصاءاته**
- **جدول تقارير سنوية** (اختياري لكنه مفيد جدًا)

ولا أوصي نهائيًا بتحويل الأوراق كما هي إلى جداول مستقلة.

---

## 12) التوصية العملية التالية
الخطوة التالية المثالية هي:
1. إنشاء قاعدة البيانات بهذا التصميم
2. كتابة سكربت استيراد من Excel إلى DB
3. إنشاء استعلام أو View يعيد إنتاج تقرير `الإجماليات`
4. لاحقًا إنشاء واجهة إدخال بدل العمل اليدوي على الإكسل
